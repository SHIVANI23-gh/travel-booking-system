Sanchari Travels - Premium Travel Booking System

An industry-level, highly responsive React & Vanilla CSS travel ticketing engine featuring visual seating customizers, persistent client-side states, and real-time query terminal handshakes.

1. Project Overview & Objectives

The objective of Sanchari Travels is to simulate a complete, realistic, and visually stunning Indian travel reservation terminal. It merges beautiful styling aesthetics with a functional system that guides a traveler from routing options all the way to seat selections and barcode-validated ticket confirmations.

Rather than building a standard static booking card form, the app adopts a sleek, modern White & Baby Pink glassmorphism visual layout and incorporates terminal sync loops (IRCTC, airports, local Volvo fleets) to showcase how asynchronous data exchange is handled.

2. Tech Stack & Architecture

The application is structured as a client-side Single Page Application (SPA) designed to build compiled production bundles extremely fast.

⚛️ React (v19)
⚡ Vite Build Tool
🎨 Vanilla CSS (Custom Variables)
🏷️ Lucide React Icons

Technology Role in Project Benefit
React 19 Component state architecture, routing, and modular views. Allows high performance reactive interfaces and structured data hooks.
Vite Bundler Local environment development server and asset building. Provides hot-module replacement and builds production bundles in < 1 sec.
Vanilla CSS Custom themes, glassmorphism filters, animations, and print media layouts. Maintains clean, custom variables, avoiding framework bloat and overrides.

3. Key Upgraded Features

Here are the premium components implemented to elevate Sanchari Travels to an industry-grade portal:

  • ⭐ Real-Time Search & Pills An instant text-input search widget filters flights, hotels, trains, and buses dynamically as you type. Includes custom-pill toggle selectors like "Breakfast Included" or "Non-stop".
  • ❤️ Wishlist & Favorites System Toggle heart buttons on listing cards to bookmark choices. Syncs to a wishlist count badge in the navbar and a dedicated Wishlist page with custom checkout triggers.
  • 🎫 Interactive Bookings Ticket Dashboard A premium dashboard showing travel stats, cancellation actions, and printable E-Tickets (boarding passes, IRCTC e-tickets, bus vouchers) with simulated barcode bars.
  • 💬 Dynamic Customer Testimonials Allows travelers to read reviews and submit their own rating testimonials via a custom validation form, updating the homepage catalog instantly.
  • 👤 Verified Customer Profile Editor Holds name, email, and phone contact data. Includes cartoon animal profile presets (bunny, panda, etc.) with real-time previews. Prefills checkout form passenger rows automatically.
  • 📱 Media Query Responsive Styles Configured fluid grids, touch-swipable seat grids for mobile compartments, and bottom navigation layouts for mobile device screens.

4. How It Was Built & Deployed

The upgrade followed a strict engineering workflow to maintain stability and prevent regression:

1. Research & Planning: Analyzed existing state variables and listing data structures in the codebase to map how bookmarks, reviews, and bookings would persist.

2. Stated Hooking: Wrote local storage persistence logic for profile parameters, favorite items arrays, and reviews. Integrated the Profile details directly into the Checkout validation fields.

3. Responsive Styling: Overrode CSS styling variables at the end of the stylesheet to support mobile wrapping, mobile tab-bar navigation, and touch-overflow elements.

4. Build Testing: Run local compiles using Vite bundler scripts to confirm the code transpiles without props errors.

5. Pushing & Deploying: Comitted the changes, pushed to the GitHub repository, and executed the deployment script (`npm run deploy`) to update the live website on the `gh-pages` branch.
